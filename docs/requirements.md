# 機材レンタル管理ソフト 要件定義

## 1. 概要

商品コード（固定形式）を軸に、機材のレンタル・返却状況を管理するデスクトップアプリケーション。
単一PCで利用し、専用ウィンドウから操作する。

## 2. 技術スタック

| 項目 | 技術 |
|---|---|
| バックエンド | Python + Flask |
| データベース | SQLite |
| フロントエンド | HTML / CSS / JavaScript |
| デスクトップ表示 | pywebview（専用ウィンドウ） |
| 配布 | PyInstaller（.exe 化） |
| 実行環境 | ローカルPC（デスクトップアプリとして動作） |

## 3. 機能要件

### 3.1 機材マスタ管理

- 商品コード（固定形式）で機材を一意に識別する
- Excelファイル（.xlsx）からの一括インポートで登録・同期する
  - インポート時は既存データと同期（追加・更新・削除を反映）
  - 非エンジニアでもExcelを編集→インポートするだけで反映可能
- 管理項目: 商品コード、商品名
- 想定件数: 約100件

### 3.2 メイン画面（1画面構成）

**すべての操作を1画面で完結させる。**

#### 検索エリア
- 商品コードを入力して検索する

#### 検索結果・操作エリア
検索後、以下の情報を表示する：

| 表示項目 | 説明 |
|---|---|
| 商品名 | 機材マスタから取得 |
| ステータス | 「レンタル中」または「貸出可能」 |
| 部署名 | レンタル者の部署（レンタル中の場合表示） |
| 氏名 | レンタル者の氏名（レンタル中の場合表示） |
| レンタル期間 | 開始日〜終了日 |

#### 状態別の動作

**貸出可能の場合：**
- 部署名・氏名・レンタル期間（開始日・終了日）を入力可能
- 「レンタル登録」ボタン → レンタルを登録
- 「返却」ボタン → **クリック不可**（disabled）

**レンタル中の場合：**
- 部署名・氏名・レンタル期間は表示のみ（**入力不可**）
- 「レンタル登録」ボタン → **クリック不可**（disabled）
- 「返却」ボタン → クリックで返却処理を実行

### 3.3 返却処理

- 返却ボタンで返却処理を実行する
- 返却後、ステータスを「貸出可能」に更新する
- 返却日時を記録する

### 3.4 レンタル履歴

- 機材ごとの過去のレンタル履歴を保持する
- 検索結果エリアの下部に履歴一覧を表示する
- 履歴には商品名・部署名・氏名・期間・ステータスを含む

## 4. データモデル

### 4.1 機材マスタ（equipment）

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | 自動採番 |
| product_code | TEXT UNIQUE | 商品コード |
| equipment_name | TEXT | 商品名 |
| created_at | DATETIME | 登録日時 |

### 4.2 レンタル履歴（rental_history）

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | 自動採番 |
| product_code | TEXT FK | 商品コード |
| equipment_name | TEXT | 商品名（履歴保持用） |
| department | TEXT | 部署名 |
| borrower_name | TEXT | 氏名 |
| rental_start | DATE | レンタル開始日 |
| rental_end | DATE | レンタル終了日 |
| status | TEXT | ステータス（レンタル中 / 返却済） |
| returned_at | DATETIME | 返却日時（返却時に記録） |
| created_at | DATETIME | 登録日時 |

## 6. 非機能要件

- 単一PC・単一ユーザー利用
- データエクスポート機能は不要
- シンプルで直感的なUI
- 非エンジニアでも機材マスタを更新できる運用設計
- `.exe` ダブルクリックで起動、ウィンドウを閉じて終了（ターミナル不要）
- 配布先PCにPythonインストール不要（.exe 単体で動作）
